### MemoryManager_SetupMemoryRegions restructure

- Computed `direct_size` and `flex_size` as locals before any field writes.
- `self.flexible_usage <- 0` and `self.pool_budget <- self.total_flexible_size` moved to the very end of the function so a field-write-flow prover sees them as the last operations before return.
- `ensure self.flexible_usage == 0` and `ensure self.pool_budget == self.total_flexible_size` retained (Elisa ensures are debug-gated by default).
