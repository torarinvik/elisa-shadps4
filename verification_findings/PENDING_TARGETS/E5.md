# E5 — Memory / Address-Space Region Invariants

## Added in this session

### core/address_space.elisa

| Function | Guards added |
|---|---|
| `AddressSpace_Map` | `requires PageAligned(virtual_addr) or virtual_addr==0`, `requires PageAligned(size) or size==0`, `requires RangeNoOverflow(virtual_addr, size)` |
| `AddressSpace_MapFile` | same three + `requires PageAligned(offset)` |
| `AddressSpace_Protect` | `requires PageAligned(virtual_addr)`, `requires PageAligned(size) or size==0`, `requires RangeNoOverflow(virtual_addr, size)` |

### core/memory.elisa

| Function | Guards added |
|---|---|
| `MemoryManager_MapMemory` | `requires PageAligned(size) or size==0`, `requires PageAligned(virtual_addr) or virtual_addr==0`, `requires RangeNoOverflow(virtual_addr, size)` |
| `MemoryManager_MapBegin` | same three |
| `MemoryManager_MapFile` | same three + `requires PageAligned(phys_addr) or phys_addr==0` |
| `MemoryManager_MapFileBegin` | same four |

### MemoryManager_SetupMemoryRegions restructure

- Computed `direct_size` and `flex_size` as locals before any field writes.
- `self.flexible_usage <- 0` and `self.pool_budget <- self.total_flexible_size` moved to the very end of the function so a field-write-flow prover sees them as the last operations before return.
- `ensure self.flexible_usage == 0` and `ensure self.pool_budget == self.total_flexible_size` retained (Elisa ensures are debug-gated by default).

## Remaining / not yet covered

- `AddressSpace_Unmap`: no page-alignment guard (size==0 early-return exists but no alignment check).
- `MemoryManager_UnmapMemory`, `MemoryManager_Protect`: no `requires` guards added (callers already validate, but guards would make it self-documenting).
- `MemoryManager_Allocate`: `AddressSpace_RangeNoOverflow` checked inline already; no top-level `requires` added.
- Cross-function `WithinReservation` postconditions on Map paths: the `InAnyReservation` check is an internal guard not yet lifted to an `ensure result` clause.
- SMT discharge of `RangeNoOverflow` on pointer arithmetic inside loops (needs `-smt` tier).
