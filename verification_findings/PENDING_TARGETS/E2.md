## Bounds analysis notes

### pfs.elisa
- `Pfs_ParseHeader` guards `data.count < 88` before any reads. All field reads go through `ff_read_*`
  which internally call `ff_can_read` and return 0 on underflow — no trap possible.
- `Pfs_ReadDirent` calls `ff_skip(reader, 512)` after 4 field reads; `ff_skip` clamps to `data.count`
  on overflow — safe but may silently consume beyond the dirent. Noted as informational.
- Validity gate (`block_size > 0 and dinode_count >= 0 and n_block >= 0`) discharges statically via
  the integer-comparison semantics; no SMT needed.

### playgo_chunk.elisa
- `Playgo_ReadHeader` unconditionally skips 32 bytes and 128 bytes via `ff_skip` (clamping). The
  `PLAYGO_HEADER_SIZE` (256) pre-check ensures the reader starts with enough data.
- `Playgo_ReadChunkAttr` and `Playgo_ReadMchunkAttr` receive an `offset` that is always checked by
  the caller (`attr_offset + PLAYGO_CHUNK_ATTR_ENTRY_SIZE > chunk_attrs_data.count` guard) before
  the call — the inner `ff_read_*` chain is safe.
- The mchunk index read (`mchunk_index_reader.offset <- attr.mchunks_offset.usize() + j*2`) can
  point past `chunk_mchunks_data`; `ff_read_u16_le` returns 0 in that case via the `ff_can_read`
  guard in `ff_read_u8`. Safe — no trap.

### npbind.elisa
- `NpBind_ReadEntry` has an explicit `ff_can_read(reader, 4)` guard before reading type+size, and
  a second `ff_can_read(reader, size.usize())` before reading the data payload. Both truncation
  paths return `NpBindEntryRaw_empty()` with `data.count == 0`.
- The body-level validity check (`body.X.data.count != body.X.size.usize()`) catches any partial
  read and sets `valid <- false`. No arithmetic overflow possible (size is u16, max 65535 bytes).
- Digest tail read: `data.count - NPBIND_DIGEST_SIZE` is only executed under `data.count >= NPBIND_DIGEST_SIZE`
  guard — no underflow.

## Bounds that could NOT be statically discharged

None. All field-read bounds are enforced operationally by `ff_can_read` / size pre-checks, and all
validity predicates are simple integer comparisons. No SMT obligation was generated that required
a `fileformat-NN.md` repro.
