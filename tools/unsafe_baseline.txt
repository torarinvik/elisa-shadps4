# Escape-hatch baseline: maximum allowed `trusted Unsafe.<Category>` sites in
# core/ + easm/. The ratchet (tools/unsafe_ratchet.sh) fails CI if any category
# exceeds its baseline or a new category appears, so the unsafe surface cannot
# silently grow. Lowering a count after removing a site is encouraged; raising
# one requires a deliberate edit here (the review signal).
#
# Format: "Unsafe.<Category> <max-count>" (comments and blank lines ignored).
Unsafe.PointerCast 24
# Boundary wrappers deliberately convert checked guest address carriers into host
# refs at the unsafe edge; each site is guarded by @boundary_pointer_args.
# 7 pthread-mutex HLEs + 8 pthread-cond HLEs + 1 TLS-origin (__tls_get_addr)
# conversion of the GuestVAddr produced by Linker_TlsGetAddr.
Unsafe.GuestHostPointerCast 16
Unsafe.MutableGlobal 10
Unsafe.AssumeProgress 9
Unsafe.UncheckedIndex 4
Unsafe.IndirectCall 3
# Runtime executable code publication is a named unsafe boundary. The single
# trusted site is the native HLE thunk publisher; callers must publish through
# GuestExec_PublishExecutableCode so debug/referee mode can track executable
# code before any indirect transfer.
Unsafe.ExecutableCodePublish 1
# Raw byte thunks are temporary machine-code builders that bypass EASM. The two
# trusted sites are the host-FS signal/exit entry shim and the native HLE
# boundary thunk; both are deliberately ratcheted until they can be moved into
# verified EASM or a checked machine-code builder contract.
Unsafe.MachineCodeBuilder 2
