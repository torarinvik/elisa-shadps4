# Escape-hatch baseline: maximum allowed `trusted Unsafe.<Category>` sites in
# core/ + easm/. The ratchet (tools/unsafe_ratchet.sh) fails CI if any category
# exceeds its baseline or a new category appears, so the unsafe surface cannot
# silently grow. Lowering a count after removing a site is encouraged; raising
# one requires a deliberate edit here (the review signal).
#
# Format: "Unsafe.<Category> <max-count>" (comments and blank lines ignored).
# One extra cast is the pthread TSD destructor used by the shadPS4-style
# per-thread Darwin LDT page. It is the explicit cleanup edge for the guest
# selector/Tcb page and should go away only if pthread key destructors gain a
# typed function-pointer ABI.
Unsafe.PointerCast 25
# Boundary wrappers deliberately convert checked guest address carriers into host
# refs at the unsafe edge; each site is guarded by @boundary_pointer_args AND a
# runtime threads_sync_boundary_guest_ptr_mapped(...) bounds check before the cast.
# All sites live in core/libraries/kernel/threads/ (pthread mutex/cond/rwlock/
# barrier/semaphore HLEs + the __tls_get_addr TLS-origin conversion of the
# GuestVAddr produced by Linker_TlsGetAddr). Count grew 16 -> 23 as more of the
# pthread-sync HLE surface was ported; each addition keeps the guarded-then-trusted
# shape, so the boundary contract is preserved.
Unsafe.GuestHostPointerCast 23
Unsafe.MutableGlobal 10
Unsafe.AssumeProgress 9
Unsafe.UncheckedIndex 4
# The extra indirect call is the host-only pthread trampoline; guest pthread
# entries still flow through the Segment.Guest TLS_CALL gate. Keeping host and
# guest entry calls separate prevents the guest segment contract from being
# smuggled into ordinary host pthreads.
Unsafe.IndirectCall 4
# Runtime executable code publication is a named unsafe boundary. The single
# trusted site is the native HLE thunk publisher; callers must publish through
# GuestExec_PublishExecutableCode so debug/referee mode can track executable
# code before any indirect transfer.
Unsafe.ExecutableCodePublish 1
# Raw byte thunks are temporary machine-code builders that bypass EASM. The two
# trusted sites are the host-FS signal/exit entry shim and the native HLE
# boundary thunk; both are deliberately ratcheted until they can be moved into
# verified EASM or a checked machine-code builder contract.
Unsafe.MachineCodeBuilder 2
# The IPC runtime input daemon is spawned detached for process lifetime via the
# blessed spawn_daemon surface. Its IpcState is heap-allocated and never freed,
# and is shared with the main thread only through a binary semaphore, so the
# cross-thread capture is lifetime- and share-safe (core/ipc/ipc.elisa).
Unsafe.ThreadShare 1
# An FFmpeg-owned opaque context pointer is cached in the module-lifetime avplayer
# handle table; FFmpeg owns the allocation until the handle is freed, so the stored
# ref is non-owning by contract
# (core/libraries/avplayer/avplayer_ffmpeg_ffi.elisa).
Unsafe.StaleRef 1
